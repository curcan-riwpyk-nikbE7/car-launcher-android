# Car Launcher — лаунчер для автомагнитолы на Android 8.0+ (CC3-style)

Кастомный домашний экран (launcher) для 2DIN автомагнитол на Android,
дизайн воссоздан по референсу лаунчера в стиле **Teyes CC3**: плеер в виде
смартфона с неоновой RGB-рамкой по контуру, карточка авто со спидометром
(MPH) на фоне неоновой дороги, белые плитки быстрых приложений, радио-виджет
с волновым визуализатором, боковая рейка с часами/ассистентом/навигацией.

## Возможности

- **Регистрируется как HOME/Launcher** — можно выбрать домашним экраном системы.
- **Плеер оформлен как "экран смартфона"** — тёмный прямоугольник с
  многоцветной неоновой рамкой по периметру (cyan → purple → pink),
  Bluetooth-бейдж, название/исполнитель реального трека, полоска
  прогресса, крупные кнопки управления снизу.
- **Работающие жесты на рабочем столе** (окончательно исправлен баг):
  - свайп влево/вправо — переключение на следующий/предыдущий трек;
  - свайп вверх/вниз — увеличение/уменьшение громкости.

  Техническая причина, почему жесты не работали раньше, и как исправлено:
  `OnTouchListener`, повешенный на `ViewGroup`, вызывается только если ни
  один дочерний View (кнопка, плитка приложения) не "забрал" ACTION_DOWN
  первым — поэтому свайп, случайно начавшийся над кликабельным элементом,
  никогда не долетал до детектора жестов. Решение — обрабатывать жесты в
  `Activity.dispatchTouchEvent()`, который видит вообще все touch-события
  ещё до раздачи по дереву View, и обязательно возвращать `true` из
  `GestureDetector.onDown()` (иначе Android прерывает разбор жеста на
  первом же касании и `onFling()` не вызывается).
- **Реальные данные плеера и статус-бара** — название трека, исполнитель и
  источник (Bluetooth/Яндекс.Музыка) берутся из `MediaSessionManager`;
  дата, статус Bluetooth/Wi-Fi и громкость — из системных API.
- **Плитки приложений больше не превращаются в белые квадраты с сырым
  именем пакета**, если ярлык по умолчанию (Яндекс.Навигатор, системное
  FM-радио и т.п.) не установлен на конкретном устройстве. Раньше
  `PackageManager` не мог отдать ни иконку, ни название для отсутствующего
  пакета — теперь `ShortcutManager` хранит заранее заданную "визитку"
  (красивая иконка + человекочитаемое название) для популярных приложений
  и использует её как fallback, а для реально установленных — берёт
  настоящую иконку/имя через `PackageManager`.
- **Настоящая 3D-модель машины (OpenGL ES 2.0)** на карточке авто —
  **третья версия, пропорции визуально проверены перед переносом в код**:
  предыдущие версии несколько раз "гадали" координаты вслепую прямо в
  Java, из-за чего колёса прятались внутри кузова, а камера смотрела под
  неудачным углом. На этот раз точная копия той же геометрии сначала
  была отрендерена в Python (matplotlib 3D) и визуально проверена с
  нескольких ракурсов (см. папку [`car_geometry_check/`](car_geometry_check/) —
  `car_4_angles.png`, `car_orbit_camera_view.png`, `render_check.py`),
  и только после подтверждения, что силуэт реально читается как машина,
  координаты перенесены в реальный рендерер:
  - колёса теперь ШИРЕ кузова (`WHEEL_X=1.0` > `BODY_HX=0.85`) и торчат
    по бокам, как у настоящего автомобиля, а не прячутся внутри;
  - светлый корпус с белыми фарами спереди и красными стоп-сигналами
    сзади — однозначно читается силуэт легковушки с любого ракурса;
  - камера зафиксирована на угле наклона ~16° (типичный автомобильный
    ракурс), вращается только азимут по кругу;
  - **колёса крутятся пропорционально реальной скорости** с GPS
    (`SpeedProvider`/`LocationManager`);
  - **честное техническое ограничение**: это НЕ фотореалистичная
    3D-модель конкретного авто — для такого нужна лицензированная
    модель в форматах glTF/GLB и тяжёлый движок рендеринга. Это
    стилизованный neon-low-poly автомобиль, зато настоящий 3D-объект
    (не гифка/картинка), лёгкий по производительности, и в этот раз
    визуально проверенный перед публикацией, а не собранный вслепую.
- **Дополнительный реальный функционал**:
  - кольцо-кнопка "ассистент" на рейке запускает голосовое распознавание;
  - кнопка навигации внизу рейки открывает Яндекс.Навигатор/Карты или
    Google Карты;
  - иконка-шестигранник на карточке авто ведёт в настройки, иконка фары
    включает/выключает фонарик камеры;
  - карточка "Radio" кликабельна и открывает системное FM-радио.
- **Добавление своих приложений на рабочий стол** — кнопка "+" открывает
  список всех установленных приложений; выбранное закрепляется как ярлык
  (`SharedPreferences`, переживает перезапуск). Долгое нажатие — удалить.
- **Экран "Все приложения"** (`AppDrawerActivity`) — все установленные
  приложения через `PackageManager` с живым поиском.
- **Экран "Настройки лаунчера"** (`SettingsActivity`) — доступ к
  уведомлениям/геолокации (с реальным статусом), тема, единицы скорости
  (MPH/км/ч), сброс рабочего стола.

## Из чего состоит

```
app/src/main/java/com/carlauncher/app/
├── MainActivity.java             — главный экран, статус-бар, жесты, ярлыки
├── ShortcutManager.java          — хранение/добавление/удаление ярлыков рабочего стола
├── ShortcutPickerActivity.java   — экран выбора приложения для "+"
├── SystemStatusHelper.java       — реальные Wi-Fi/Bluetooth/громкость/дата
├── AppDrawerActivity.java        — экран "Все приложения" с поиском
├── SettingsActivity.java         — настройки лаунчера
├── AppItem.java / AppLauncherUtil.java — запуск приложений по пакету
└── media/
    ├── MediaRepository.java         — единая точка правды "что сейчас играет"
    ├── NotificationListener.java    — слушает все MediaSession в системе
    └── NotificationAccessUtil.java  — проверка/запрос разрешения
```

## Важные технические ограничения (актуально для реальных магнитол)

1. **Скорость "0 MPH"** — статичный дизайн-элемент. Для реальной скорости
   нужен GPS (`LocationManager`) или вендорский SDK магнитолы для чтения
   CAN-шины.
2. **Радио-виджет** — UI-заглушка с визуализатором. Реальное управление
   тюнером требует вендорского API конкретной магнитолы.
3. **Bluetooth-захват музыки** работает "из коробки" на любом Android 8+
   через официальный публичный API (`NotificationListenerService` +
   `MediaSessionManager`).

## Сборка APK

```bash
export JAVA_HOME=/path/to/jdk-21
export ANDROID_HOME=/path/to/android-sdk
./gradlew assembleDebug     # debug-сборка
./gradlew assembleRelease   # release-сборка, подписана тестовым keystore
```

Готовые собранные копии лежат прямо в репозитории:
- [`dist/CarLauncher-debug.apk`](dist/CarLauncher-debug.apk)
- [`dist/CarLauncher-release.apk`](dist/CarLauncher-release.apk)

### Автосборка через GitHub Actions

При каждом push в `main` запускается [`.github/workflows/build.yml`](.github/workflows/build.yml),
который собирает debug и release APK и прикладывает их как build-артефакты.

## Установка на магнитолу

1. Скопируйте `CarLauncher-release.apk` на магнитолу (флешка / ADB).
2. Установите APK.
3. Нажмите кнопку "Домой" на магнитоле → выберите Car Launcher → "Всегда".
4. При первом запуске разрешите доступ к уведомлениям (для чтения Bluetooth/Яндекс.Музыки).
5. Нажмите "+" в блоке "Application", чтобы добавить свои ярлыки на рабочий стол.
